## Análise do Motor Gemma 4
O objeto `JiraIssue` possui uma estrutura aninhada complexa e potencialmente redundante dentro de `fields.description`. A estrutura atual é excessivamente aninhada (`content?: Array<{ content?: Array<{ text: string; } }>>`), dificultando a fácil manipulação e a renderização da descrição do Jira (que geralmente é um bloco de texto simples ou Markdown). Além disso, a forma como as propriedades `onSelectIssue` são definidas (`{ key: string; summary: string; description: string }`) exige que a descrição seja tratada como uma string simples, o que contrasta com o tipo complexo definido no `JiraIssue`. Esta refatoração visa simplificar a tipagem e garantir que a camada de apresentação (UI) receba os dados no formato mais utilizável possível, melhorando a manutenibilidade e a clareza da aplicação.
