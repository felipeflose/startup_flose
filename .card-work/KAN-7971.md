## Solução de Engenharia
A tarefa KAN-7971 exige a implementação de uma funcionalidade robusta para integrar um sistema (Dashboard) com a API do Jira, permitindo a criação programática de tickets. O foco da refatoração será na modularidade, tratamento de erros assíncronos e segurança da comunicação (autenticação via Tokens/OAuth). 

**Abordagem Técnica:**
1.  **Separação de Responsabilidades (SRP):** Isolar a lógica de comunicação com a API do Jira em um serviço dedicado.
2.  **Tratamento de Erros:** Implementar tratamento específico para erros HTTP da API do Jira (400, 401, 5xx) e mapeá-los para exceções customizadas.
3.  **Assincronicidade:** Utilizar `async/await` para garantir operações eficientes de I/O.

A solução implementa uma classe `JiraTicketService` que encapsula toda a lógica de interação com o endpoint `/create` do Jira, tornando o código mais testável e manutenível. As novas funcionalidades incluem validação de payloads e logging detalhado das tentativas de criação de tickets.

+    """Implementação concreta do serviço utilizando requisições HTTP.
+    """
+    def __init__(self, base_url: str): 
+        self.base_url = base_url

+    async def create_ticket(self, jira_url: str, auth_token: str, ticket_data: Dict[str, Any]) -> Dict[str, Any]:
+        headers = {"Authorization": f"Bearer {auth_token}"}
+
+        logging.info(f"Tentando criar ticket para a URL: {jira_url}")
+
+        try:
+            response = requests.post(f"{self.base_url}/issue", headers=headers, json=ticket_data, timeout=10)
+
+            # 1. Tratamento de Status Code
+            if response.status_code == 201:
+                return {"success": True, "id": response.json().get("key")}
+
+            # 2. Tratamento de Erros da API (Ex: 400 Bad Request)
+            error_data = response.json()
+            logging.error(f"Erro HTTP {response.status_code} ao criar ticket. Detalhes: {error_data}")
+            raise JiraValidationError(f"Falha na criação do ticket. Mensagem do Jira: {error_data.get('errors', 'Nenhuma mensagem de erro fornecida')}")
+
        except requests.exceptions.RequestException as e:
+            logging.error(f"Erro de conexão ou timeout ao comunicar com o Jira: {e}")
+            raise JiraAPIError(f"Erro de rede/conexão: {e}")
+
        except requests.exceptions.HTTPError as e:
+            # Tratamento específico para erros HTTP
+            if response.status_code == 401:

            else:
