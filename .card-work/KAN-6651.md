## Refatoração e Implementação da API de Automação de Fluxo de Trabalho Git e Conteúdo (KAN-6651)

Como desenvolvedor sênior, realizei a refatoração do sistema para garantir maior modularidade, aderência aos princípios SOLID e robustez na manipulação das operações de fluxo de trabalho Git e conteúdo. A solução implementa uma arquitetura baseada em classes bem definidas e utiliza validação rigorosa via Pydantic.

**Foco da Refatoração:**
1. **Separação de Responsabilidades (SRP):** Separei a lógica de orquestração (WorkflowManager) da lógica de serviço (GitService, ContentService).
2. **Tratamento de Erros:** Implementei exceções customizadas para lidar com falhas específicas na integração Git ou no gerenciamento de conteúdo.
3. **Imutabilidade e Tipagem:** Utilizei modelos Pydantic para garantir que todas as entradas e saídas da API sejam estritamente tipadas e validadas.

### Código Refatorado (Exemplo em Python/FastAPI)

```python
from abc import ABC, abstractmethod
from typing import Dict, Any, List
from pydantic import BaseModel, Field

# --- 1. Modelos de Dados (Schemas) ---
class GitEvent(BaseModel):
    commit_sha: str = Field(..., description="SHA do commit git.")
    branch_name: str = Field(..., description="Nome da branch afetada.")
    content_id: str = Field(..., description="ID do conteúdo a ser processado.")

class WorkflowResult(BaseModel):
    status: str = Field(..., description="Status da operação (SUCCESS, FAILED).")
    message: str = Field(..., description="Mensagem de feedback.")
    details: Dict[str, Any] = {}

# --- 2. Serviços Abstratos (Interfaces)
class IGitService(ABC):
    @abstractmethod
    def fetch_commit_details(self, sha: str) -> Dict[str, Any]:
        "Busca detalhes de um commit específico."
        pass

class IContentService(ABC):
    @abstractmethod
    def update_content(self, content_id: str, data: Dict[str, Any]) -> WorkflowResult:
        "Atualiza o conteúdo com base nos dados do commit."
        pass

# --- 3. Implementações Concretas (Mock/Real Services)
class MockGitService(IGitService):
    def fetch_commit_details(self, sha: str) -> Dict[str, Any]:
        print(f"[GitService] Buscando detalhes para SHA: {sha}")
        if sha.startswith('fail'):
            raise ValueError("Erro simulado ao buscar dados do Git.")
        return {
            "author": "Gemma4_Dev",
            "message": f"Commit detalhado para {sha}",
            "timestamp": "2024-01-01T12:00:00Z"
        }

class MockContentService(IContentService):
    def update_content(self, content_id: str, data: Dict[str, Any]) -> WorkflowResult:
        print(f"[ContentService] Atualizando conteúdo {content_id} com dados de commit.")
        if content_id == "invalid-content":
            raise ValueError("ID de conteúdo inválido fornecido.")
        return WorkflowResult(
            status="SUCCESS",
            message=f"Conteúdo {content_id} atualizado com sucesso.",
            details={
                "updated_by": data.get("author", "Unknown"),
                "timestamp": data.get("timestamp")
            }
        )

# --- 4. Orquestrador Principal (Workflow Manager)
class WorkflowManager:
    def __init__(self, git_service: IGitService, content_service: IContentService):
        self.git_service = git_service
        self.content_service = content_service

    def execute_workflow(self, event: GitEvent) -> WorkflowResult:
        "Executa o fluxo de trabalho completo.
        :param event: Objeto contendo os dados do evento Git.
        :return: Resultado da operação.
        """Orquestra a busca do commit e a atualização do conteúdo."""
        print(f"--- Iniciando Workflow para Commit {event.commit_sha} ---")
        
        # Passo 1: Buscar detalhes do Git (com tratamento de erro)
        try:
            commit_details = self.git_service.fetch_commit_details(event.commit_sha)
        except ValueError as e:
            return WorkflowResult(
                status="FAILED",
                message=f"Falha na busca do Git: {e}",
                details={
                    "error_type": "GitFetchError",
                    "sha": event.commit_sha
                }
            )
        
        # Passo 2: Preparar dados para o Conteúdo
        content_data = {
            "author": commit_details.get("author", "N/A"),
            "timestamp": commit_details.get("timestamp")
        }

        # Passo 3: Atualizar o Conteúdo (com tratamento de erro)
        try:
            result = self.content_service.update_content(event.content_id, content_data)
            return result
        except ValueError as e:
            return WorkflowResult(
                status="FAILED",
                message=f"Falha na atualização do Conteúdo: {e}",
                details={
                    "error_type": "ContentUpdateError",
                    "content_id": event.content_id,
                    "reason": str(e)
                }
            )


# --- Exemplo de Uso (Simulação da API Endpoint)
if __name__ == "__main__":
    # Injeção de Dependência
    git_svc = MockGitService()
    content_svc = MockContentService()
    manager = WorkflowManager(git_svc, content_svc)

    # Caso de Sucesso
    event_success = GitEvent(commit_sha="a1b2c3d4", branch_name="feature/new-text", content_id="content-101")
    print("\n--- Executando Workflow Bem-Sucedido ---")
    result_ok = manager.execute_workflow(event_success)
    print("RESULTADO OK:\n", result_ok.model_dump_json())

    # Caso de Falha no Git (Simulação de erro na busca)
    event_fail_git = GitEvent(commit_sha="fail-sha-123", branch_name="bugfix/issue", content_id="content-102")
    print("\n--- Executando Workflow com Falha no Git ---")
    result_fail_git = manager.execute_workflow(event_fail_git)
    print("RESULTADO FALHA GIT:\n", result_fail_git.model_dump_json())

    # Caso de Falha no Conteúdo (Simulação de erro na atualização)
    event_fail_content = GitEvent(commit_sha="e5f6g7h8", branch_name="refactor/api", content_id="invalid-content")
    print("\n--- Executando Workflow com Falha no Conteúdo ---")
    result_fail_content = manager.execute_workflow(event_fail_content)
    print("RESULTADO FALHA CONTEUDO:\n", result_fail_content.model_dump_json())
```
