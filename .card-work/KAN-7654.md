## Solução de Engenharia
A implementação do componente de dashboard interativo para integração Jira requer uma refatoração focada em três pilares: Separação de Responsabilidades (SoC), tratamento assíncrono robusto e melhor gerenciamento do estado (State Management). A solução proposta foca em encapsular a lógica de comunicação com a API da Jira em um serviço dedicado, garantindo que o componente de UI seja puramente responsável pela apresentação dos dados. Refatoraremos o código para utilizar *hooks* personalizados e garantir a imutabilidade dos dados ao manipular as chamadas de API, tornando o código mais testável e escalável.

function JiraDashboard({ projectId }) {
  // 1. Abstração: Utiliza o serviço injetado em vez de chamadas diretas a fetch.
  const { jiraData, loading, error } = useJiraService(projectId);

  if (loading) {
    return <LoadingSpinner message="Carregando dados do Jira..." />;
  }

  if (error) {
    return <ErrorDisplay message={`Erro ao carregar: ${error.message}`} retryAction={() => {/* Lógica de Retry */}} />;
  }

  if (!jiraData || jiraData.length === 0) {
    return <EmptyState message="Nenhum dado encontrado para este projeto." />;
  }

  // 2. Apresentação: O componente agora é puramente responsável por renderizar.
  return (
    <DashboardContent data={jiraData} project={projectId} />
  );
}
