A função `createGitBranch` utiliza `child_process.exec` para executar comandos Git, o que introduz riscos de segurança (shell injection) e complexidade na gestão de erros. Esta abordagem é frágil e não otimizada para ambientes Node.js modernos. É necessário refatorar esta função para utilizar a API mais segura do módulo `child_process`, como `execFile` ou `spawn`, garantindo melhor isolamento, controle de entrada e tratamento robusto de erros.
