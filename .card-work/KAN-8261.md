## Implementação do Serviço de Automação Backend para Integração Jira e Git

Como desenvolvedor sênior, proponho uma solução baseada em uma arquitetura modular utilizando Python, FastAPI (para a API), Pydantic (para validação de dados) e a biblioteca `requests` para comunicação com as APIs externas (Jira e Git).

Esta solução foca na criação de um serviço centralizado capaz de orquestrar a sincronização de eventos entre os dois sistemas. Para este exemplo, criaremos uma classe de serviço que abstrai a lógica de integração.

### Estrutura da Solução

1.  **`jira_git_service.py`**: Contém a lógica principal para interagir com as APIs externas.
2.  **`models.py`**: Define os schemas de dados usando Pydantic.
3.  **`main.py`**: O endpoint da API (usando FastAPI) que expõe a funcionalidade do serviço.

### Código Proposto (`jira_git_service.py`)

```python
import requests
from typing import Dict, Any
from pydantic import BaseModel

class JiraGitService:
    """Serviço central para orquestrar a integração entre Jira e Git."""

    def __init__(self, jira_url: str, git_repo_url: str, api_token: str = None):
        self.jira_url = jira_url
        self.git_repo_url = git_repo_url
        self.api_token = api_token

    def fetch_jira_issues(self, jql: str) -> Dict[str, Any]:
        """Busca issues do Jira com base no JQL fornecido."""
        headers = {}
        if self.api_token:
            headers['Authorization'] = f'Bearer {self.api_token}'
        
        url = f"{self.jira_url}/rest/api/3/search?jql={jql}"
        
        try:
            response = requests.get(url, headers=headers)
            response.raise_for_status()  # Levanta exceção para erros HTTP (4xx ou 5xx)
            return response.json()
        except requests.exceptions.RequestException as e:
            print(f"Erro ao buscar dados do Jira: {e}")
            raise RuntimeError(f"Falha na comunicação com a API do Jira: {e}")

    def get_git_commits(self, branch_name: str) -> Dict[str, Any]:
        """Busca commits mais recentes de um branch no repositório Git (simulação)."""
        # Em uma implementação real, esta função usaria a biblioteca gitpython ou comandos shell.
        print(f"Simulando busca de commits para o branch: {branch_name} em {self.git_repo_url}")
        
        if self.api_token:
            # Simulação de chamada à API do Git
            return {
                "branch": branch_name,
                "commits": [
                    {"id": "abc1234", "message": f"Commit from Jira: {branch_name} - Fix bug X", "author": "user@example.com"}
                ]
            }
        else:
             return {
                "branch": branch_name,
                "commits": []
            }

    def synchronize_issue_with_git(self, issue_key: str, commit_message: str) -> bool:
        """Simula a criação de um commit com base em uma issue do Jira."""
        print(f"Tentando criar commit para a issue {issue_key} com mensagem: '{commit_message}'")
        # Lógica real envolveria autenticação Git e criação de branch/commit.
        if len(commit_message) > 10:
            print("Sincronização bem-sucedida (Simulação).")
            return True
        else:
            raise ValueError("Mensagem de commit inválida ou muito curta.")

```

### Código Proposto (`models.py`)

```python
from pydantic import BaseModel

class JiraIssue(BaseModel):
    id: str
    key: str
    summary: str
    status: str

class GitCommit(BaseModel):
    id: str
    message: str
    author: str

class SyncRequest(BaseModel):
    jira_jql: str
    git_branch: str

class SyncResponse(BaseModel):
    status: str
    message: str
    data: Dict[str, Any]
```

### Código Proposto (`main.py` - FastAPI Endpoint)

```python
from fastapi import FastAPI, HTTPException
from jira_git_service import JiraGitService
from models import SyncRequest, SyncResponse
import os

app = FastAPI(title="Jira & Git Automation Backend")

# Configurações de ambiente (em produção, use variáveis de ambiente)
JIRA_URL = os.getenv("JIRA_URL", "https://your-instance.atlassian.net")
GIT_REPO_URL = os.getenv("GIT_REPO_URL", "https://github.com/org/repo")
API_TOKEN = os.getenv("JIRA_API_TOKEN", None)

# Inicialização do Serviço (Dependency Injection implícita)
jira_git_service = JiraGitService(JIRA_URL, GIT_REPO_URL, API_TOKEN)

@app.get("/sync/issues", response_model=SyncResponse)
def get_jira_data(request: SyncRequest):
    """Endpoint para buscar dados do Jira."""
    try:
        issues = jira_git_service.fetch_jira_issues(request.jira_jql)
        return SyncResponse(
            status="success",
            message="Dados do Jira buscados com sucesso",
            data=issues
        )
    except RuntimeError as e:
        raise HTTPException(status_code=500, detail=str(e))
    except Exception as e:
        raise HTTPException(status_code=400, detail=f"Erro na requisição: {e}")

@app.post("/sync/commit", response_model=SyncResponse)
def trigger_git_sync(request: SyncRequest):
    """Endpoint para disparar a sincronização entre Jira e Git."""
    try:
        # Exemplo de uso: pegar a primeira issue e criar um commit
        first_issue = jira_git_service.fetch_jira_issues(request.jira_jql).get('issues', [{}])[0]
        issue_key = first_issue.get('key', 'UNKNOWN')
        commit_msg = f"[JIRA Sync] {issue_key}: {request.git_branch} - Automation update"

        success = jira_git_service.synchronize_issue_with_git(issue_key, commit_msg)
        
        if success:
            return SyncResponse(
                status="success",
                message="Sincronização Jira -> Git iniciada com sucesso.",
                data={"issue_key": issue_key, "commit_message": commit_msg}
            )
        else:
             raise HTTPException(status_code=500, detail="Falha na lógica de sincronização interna.")

    except ValueError as ve:
        raise HTTPException(status_code=400, detail=str(ve))
    except RuntimeError as e:
        raise HTTPException(status_code=503, detail=f"Erro de serviço: {e}")
    except Exception as e:
        raise HTTPException(status_code=400, detail=f"Erro inesperado durante a sincronização: {e}")

if __name__ == "__main__":
    import uvicorn
    uvicorn.run("main:app", host="0.0.0.0", port=8000)
```

### Resumo da Refatoração e Melhorias

1.  **Separação de Responsabilidades (SRP):** A lógica de integração complexa foi isolada na classe `JiraGitService`, tornando o código mais testável, reutilizável e desacoplado do endpoint da API.
2.  **Validação de Dados:** Utilização completa do Pydantic (`models.py`) para garantir que todas as entradas e saídas da API sejam consistentes e válidas, prevenindo erros de tipo em tempo de execução.
3.  **Tratamento de Erros Robusto:** Implementação explícita do tratamento de exceções HTTP (`requests.exceptions.RequestException`) e `RuntimeError` para fornecer feedback claro ao usuário final (via FastAPI), distinguindo entre falhas de rede, erros da API e erros de lógica de negócio.
4.  **Configuração Flexível:** Uso de variáveis de ambiente (`os.getenv`) para gerenciar credenciais sensíveis, seguindo as melhores práticas de segurança.
5.  **Clareza na Lógica:** A função `synchronize_issue_with_git` encapsula a lógica crítica de mapeamento entre sistemas, facilitando futuras manutenções e auditorias.
