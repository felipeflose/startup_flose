## Solução de Engenharia
A tarefa KAN-7705 exige a implementação de um serviço robusto de gerenciamento de fluxo de trabalho que sincroniza eventos do Git com o Jira. O foco da solução é refatorar a lógica existente para melhorar a modularidade, adicionar tratamento de erros robusto (especialmente ao interagir com APIs externas) e garantir a idempotência das operações. 

**Estratégia de Refatoração:**
1. **Separação de Responsabilidades (SRP):** Isolar a lógica de comunicação com o Git da lógica de comunicação com o Jira.
2. **Padrão de Projeto:** Implementar um padrão de Repositório para abstrair as chamadas HTTP/API do Jira e do Git.
3. **Tratamento de Erros:** Implementar *try-catch* específicos para falhas de autenticação, limites de taxa (rate limiting) e erros de validação da API.
4. **Assincronicidade:** Utilizar filas ou *workers* para processar eventos de forma assíncrona, evitando bloqueios na interface do usuário ou no fluxo principal.

**Implementação Proposta:** Criaremos um serviço central (`WorkflowIntegrationService`) que orquestra as chamadas, delegando a comunicação específica para classes dedicadas (e.g., `GitHandler`, `JiraClient`).
