## Análise do Motor Gemma 4
A função `createGitBranch` utiliza `child_process.exec()` com o argumento `branchName`, que presumivelmente é um input vindo do usuário (ou de outro serviço chamador). A utilização direta desse input dentro de uma string shell (`git checkout -b ${branchName}`) sem qualquer sanitização ou validação rigorosa cria uma vulnerabilidade grave de Injeção de Comando (Command Injection). Um atacante mal-intencionado pode fornecer um valor para `branchName` como, por exemplo: `malicioso'; rm -rf /tmp/dados; #`. Isso permitiria a execução remota de código arbitrária (RCE) no servidor que hospeda o processo Node.js. É imperativo migrar esta funcionalidade para utilizar APIs do módulo `child_process` que aceitem argumentos em formato array, eliminando a necessidade de construir comandos via string shell, ou implementar uma validação estrita (regex) no input `branchName`.

**Critérios de Aceitação:** 1. O processo deve usar `spawn()` ou `execFile()` em vez de `exec()`. 2. Deve ser aplicada uma função de sanitização que limite o nome do branch a caracteres alfanuméricos e hífens (`^[a-zA-Z0-9-]+$`). 3. Em caso de falha na execução (e não apenas no checkout), a exceção deve ser tratada de forma mais granular, reportando o erro sem expor detalhes internos do sistema operacional.
