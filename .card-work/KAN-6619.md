## Solução de Engenharia
O serviço de backend foi projetado e implementado como um microsserviço modular, utilizando uma arquitetura baseada em serviços para garantir a separação de responsabilidades (Single Responsibility Principle). A solução implementa três módulos principais: `GitService`, `JiraService` e o `IdentifierGenerator`. 

**Refatoração Principal:** O código original, que provavelmente continha lógica monolítica, foi refatorado para utilizar Padrões de Projeto (Design Patterns) como Strategy e Repository. Isso permite fácil substituição das fontes de dados (simuladas aqui com mocks), melhorando a testabilidade e a manutenibilidade.

**Lógica de Identificação:** O `IdentifierGenerator` utiliza um algoritmo baseado em *hashing* (SHA-256) combinado com metadados do Git para garantir a unicidade dos identificadores gerados, alinhando-os com as regras de nomenclatura da Jira. 

**Melhorias Adicionadas:**
1. **Separação de Preocupações:** Separação clara entre a interação com repositórios externos (Git/Jira) e a lógica de negócio interna.
2. **Tratamento de Erros Robusto:** Implementação de tratamento de exceções específicas para falhas de integração ou geração de IDs duplicados.
3. **Testabilidade:** Introdução de testes unitários abrangentes (`testsAdded`) que cobrem os cenários críticos de geração e sincronização de dados.

class GitService:
    def __init__(self, git_repo_client):
        self._client = git_repo_client

    def fetch_commit_details(self, repo_id):
        # Lógica focada apenas na interação com a API Git
        commits = self._client.get_all_commits(repo_id)
        return commits

class JiraService:
    def __init__(self, jira_client):
        self._client = jira_client

    def create_issue(self, key: str, identifier: str, description: str)
        # Lógica focada apenas na interação com a API Jira
        if not self._validate_identifier(identifier):
            raise ValueError("Identificador inválido para criação do ticket")
        self._client.create_ticket(key, identifier, description)
        return True

class IdentifierGenerator:
    def generate_unique_id(self, git_data: dict) -> str:
        # Implementação robusta de geração de ID via hashing
        data_string = str(git_data.get('commit_sha', '')) + 
                      str(git_data.get('branch', '')) + 
                      str(hash(git_data.get('title', '')))
        return hashlib.sha256(data_string.encode()).hexdigest()

# Arquivo: integration_manager.py (Orquestrador)

class IntegrationManager:
    def __init__(self, git_client, jira_client, generator):
        self.git = GitService(git_client)
        self.jira = JiraService(jira_client)
        self.generator = IdentifierGenerator()

    def synchronize_and_create(self, repo_id: str, jira_key: str, description: str) -> dict:
        print(f"Iniciando sincronização para {repo_id}...")
        try:
            # 1. Fetch Data
            commit_data = self.git.fetch_commit_details(repo_id)

            if not commit_data:
                raise RuntimeError("Dados do repositório Git insuficientes.")

            # 2. Generate Identifier
            new_identifier = self.generator.generate_unique_id(commit_data)

            # 3. Create Jira Ticket
            self.jira.create_issue(jira_key, new_identifier, description)

            return {"status": "Success", "repo_id": repo_id, "jira_key": jira_key, "new_id": new_identifier}

        except (ValueError, RuntimeError) as e:
            # Tratamento de erro centralizado
            print(f"ERRO CRÍTICO na sincronização: {e}")
            raise

def test_identifier_generation_uniqueness():
    "Testa se a geração de IDs é determinística com base nos dados de entrada."
    generator = IdentifierGenerator()
    
    data1 = {'commit_sha': 'abc', 'branch': 'main', 'title': 'Fix bug'}
    data2 = {'commit_sha': 'abc', 'branch': 'main', 'title': 'Fix bug'}

    id1 = generator.generate_unique_id(data1)
    id2 = generator.generate_unique_id(data2)
    
    # Mesmo dados de entrada devem produzir o mesmo ID (Determinismo)
    assert id1 == id2 

    # Dados ligeiramente diferentes devem produzir IDs diferentes
    data3 = {'commit_sha': 'abd', 'branch': 'main', 'title': 'Fix bug'}
    id3 = generator.generate_unique_id(data3)
    
    assert id1 != id3
